🔗 LATTICE

Accountability Layer for Multi-Agent AI Systems
Ali Murtaza Bhutto | ORCID: 0009-0007-2787-943X | MIT License | March 2026

⚠️ The Problem

Multi-agent AI systems produce conclusions but can't explain how they got there. When your pipeline says "this domain is malicious," there's no way to:

  • Trace that conclusion back to raw evidence
  • Verify nothing was fabricated or modified
  • Identify which agent made each assertion
  • Assess what happens if one data point is wrong

No existing agent framework (LangGraph, CrewAI, AutoGen) provides standardized accountability infrastructure.

✅ The Solution

LATTICE records every agent decision as a content-addressed, cryptographically signed Claim in a Directed Acyclic Graph.

  • SHA-256 hashing makes claims tamper-evident
  • Ed25519 signatures bind claims to agents
  • DAG structure connects conclusions to evidence
  • Confidence tracking on every assertion

Not an agent framework. An accountability layer that sits underneath any framework.

🔍 Demo: OSINT Investigation Trace

Three agents investigate a suspicious domain. Every finding is recorded. The final report traces back to raw tool output:

⚡ How It Works

  • Agent registers with auto-generated Ed25519 keypair
  • Raw evidence stored as content-addressed blob
  • Agent creates claim referencing evidence by hash
  • Claim hashed (SHA-256 of canonical JSON)
  • Claim signed (Ed25519 over the hash)
  • Stored in SQLite DAG
  • Trace/Audit/Verify at any time

💡 Why It Matters

  • Security consulting: clients need auditable reasoning, not "the AI said so"
  • OSINT/Journalism: Bellingcat-grade source attribution for automated pipelines
  • EU AI Act: explainability requirement for high-risk AI systems
  • Legal proceedings: provenance is everything in digital evidence
  • Research: reproducible, inspectable investigative methodology
